# NV36 (A)

## Answer

FM forms should match the design system by adhering to the following guidelines:

- **Use Tokens**: Ensure that all form elements utilize design tokens from `Navorina/css/variables.css` instead of hardcoded values.
- **Consistent Heights**: Maintain a uniform height for all input fields, as specified by `--control-height-md` for desktop and `--control-height-mobile` for mobile.
- **Shared Layout**: Follow the shared layout rules by copying header, footer, and navigation from the established design patterns.
- **Form Elements**: Implement forms according to the specifications in `docs/design-system/COMPONENT_LIBRARY.md`, ensuring consistency in styles and behavior.
- **Accessibility**: Ensure that all forms are accessible, with visible focus on interactive elements and appropriate use of `aria-*` attributes.
- **Mobile Considerations**: Design forms to be responsive, following the guidelines for mobile layouts as outlined in the design system.

By following these principles, FM forms will align with the overall design system and provide a cohesive user experience.
