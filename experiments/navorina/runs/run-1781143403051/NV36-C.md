# NV36 (C)

## Answer

To ensure FM forms match the design system, the following guidelines should be followed:

- **Use Tokens**: Styles should be based on tokens defined in `Navorina/css/variables.css`.
- **Consistent Height**: Ensure all forms have a uniform height.
- **Select Elements**: Use select elements with checkmarks for better user experience.
- **Date Icons**: Incorporate date icons in date input fields.
- **Accessibility**: Implement `aria-*` attributes on critical buttons for better accessibility.
- **Progressive Disclosure**: Follow the principle of progressive disclosure to avoid overwhelming users.
- **No Dashboard Overload**: Avoid cluttering the UI with excessive information or features.

These steps align with the UX principles outlined in the design system documentation.
